FAQ
Commonly asked questions related to cohort analysis in WebEngage
1. Can I download a report?
Currently, the WebEngage dashboard does not allow users to download an analysis report. However, the feature is in the pipeline and will go live on your dashboard pretty soon!
2. Can I select different time frames for the
First Event
and the
Return Event?
While it’s not possible to select a time frame per event, you can choose a custom period through the
date range filter
(placed on the top right corner of the section) within which the users have performed both, the
First Event
and the
Return Event
. You can then choose to analyze the cohort in different formats of time, namely -
Days/Weeks/Months.
3. Can I select different time formats for
First Event
and
Return Event?
Currently, the WebEngage dashboard does not extend the functionality of analyzing the repeat behavior of cohorts over various time frames. You can analyze it in
Days/Weeks/Months
- with only one format reflected across the table at any given point in time.
4. Can I create and save a list of cohorts to track behavior over a long period?
Currently,
Cohorts
is a primary, transactional section in your dashboard. This means that you can analyze as many behavioral trends as you like! However, you will not be able to create a list of defined cohorts and refer to them on a later date. Don’t be disheartened. Our tech geniuses are working on it, and the functionality will soon be available.
5. Can I add custom parameters to the field
Split By
, for my analysis?
All the parameters listed under the field,
Split By
in the query bar, are currently un-editable and have been predefined by us. It includes all the system attributes automatically gleaned by us for each event,

.

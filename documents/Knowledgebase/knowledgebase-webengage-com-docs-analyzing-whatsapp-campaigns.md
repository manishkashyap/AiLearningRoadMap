Why is the User ID shown multiple times?
As visible in the above visual, on searching for a user, multiple rows get populated with the same
User ID.
Each row contains a different message status with a varying date-time of
Delivery, Read, and Clicks.
This is because the same user was sent the
recurring WhatsApp campaign
in multiple runs, and in each run, they interacted differently with the message. Thus, if a triggered, recurring, transactional, journey, or relay campaign is sent to a user multiple times, then you will be able to analyze their interaction with each one.

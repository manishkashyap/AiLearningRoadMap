Service Partner Integration
Getting Started
How to add your ESP, SSP, WSP and RSP as a partner to the WebEngage User Retention Suite
Although WebEngage offers a plethora of service providers for message delivery, if you are not using one of these, you can request your service provider to add an integration for WebEngage. As a result, your service provider will then also have exposure to the 40,000+ clients of WebEngage.
Service providers will consume the WebEngage API explained here to receive the messages and to send the delivery notification to WebEngage in the defined format.
How It Works
Service provider builds the integration to consume messages from WebEngage and to forward the delivery notification to WebEngage. This is shown in the flowchart below.
Service provider shares the corresponding API with WebEngage, which is then verified for quality by WebEngage.

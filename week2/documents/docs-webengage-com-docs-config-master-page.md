Getting Your App Ready for WebEngage
Master Configuration Page
To use WebEngage features such as Push Notifications, In-app messages, SMS, Email, WhatsApp, RCS, and On-site messaging, your app must first be integrated with the WebEngage SDK.
App integration enables WebEngage to track user activity, manage user profiles, and deliver messages across channels reliably.
When is app integration required?
You must complete app integration if you plan to:
Send Push or In-app messages
Track user events and user attributes
Create segments and journeys
Personalize communication across any WebEngage channel
If your app is already integrated, you can directly proceed with configuring the required channel

What happens after integration?
Once your app is successfully integrated with WebEngage, no additional app-level setup is required to enable new channels. You can start your integrations
Data Platform> Integrations> Channel Integration Status> Channel Setup (Configure)> Your choice of channel of your dashboard to enable the channel.
